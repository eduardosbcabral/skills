# Program Brief

Use for normal/risky idea work. Keep it short enough to remain a working model.

```text
Source facts:
Assumptions:
Contradictions:
Unknowns:
Decisions needed:

Actors and workflow:
Terms and rules:
Permissions and lifecycle:
Data and integrations:
Constraints:

First vertical slice:
User/system outcome:
Scope / out of scope:
Acceptance examples:
Objective verifier:
Dependencies and risks:
Deferred decisions:
```

A good first slice crosses the necessary layers, delivers one observable outcome, and does not build infrastructure for hypothetical later slices.

Stop for user decisions involving money, auth, permissions, tenant/data ownership, destructive operations, legal/compliance/privacy, or ambiguous core workflow semantics.
