# Batch Grill Me

Use only when a small set of unresolved decisions blocks implementation.

1. Read relevant docs, domain language, tests, and current behavior first.
2. List independent, currently answerable decisions as the **decision frontier**.
3. Ask the frontier in one batch. For each question include why it matters, 2–3 real options, and a recommended answer with its tradeoff.
4. Hold dependent questions for the next round.
5. Reconcile answers with evidence and expose contradictions.
6. Stop when acceptance behavior and implementation boundaries are settled.

Do not ask the user for facts the repository or runtime can answer. The user owns behavior and business decisions.
