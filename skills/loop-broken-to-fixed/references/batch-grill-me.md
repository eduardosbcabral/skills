# Batch Grill Me

Use only when expected behavior or risky fix semantics are genuinely unclear. Do not use it to replace diagnosis.

1. Research docs, tests, current behavior, artifacts, and runtime evidence first.
2. Form the **decision frontier** from independent semantic decisions that block a safe fix.
3. Ask them in one batch. For each question include why it matters, 2–3 real options, and a recommended answer with its tradeoff.
4. Hold dependent questions for the next round.
5. Reconcile answers with evidence; expose contradictions.
6. Stop when expected behavior is settled, then return to diagnosis.

Facts about the failure are sensor questions, not user interview questions.
